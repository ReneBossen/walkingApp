- `List.Item` (settings items)
- `Switch` (toggle notifications, features)
- `RadioButton` (single choice selections)
- `Slider` (step goal adjustment)
- `Modal` / `Portal` (pickers and dialogs)
- `Button` (save, sign out)
- `Divider` (section separators)
- `Dialog` (confirmations)

## API Calls

```typescript
GET /api/users/me // Get current user with preferences
PUT /api/users/me/preferences // Update preferences
POST /api/users/me/change-password // Change password
POST /api/auth/signout // Sign out (invalidate session)
GET /api/push-tokens // Get registered push tokens
POST /api/push-tokens // Register push token
DELETE /api/push-tokens/{token} // Remove push token
```

- `showUnitsModal: boolean`

- `showThemeModal: boolean`
- `selectedUnits: 'metric' | 'imperial'`
- `selectedGoal: number`
- `selectedTheme: 'light' | 'dark' | 'system'`

**Global State:**
```typescript
interface UserStore {
  preferences: UserPreferences;
```

## Interactions

1. **Tap Profile**: Navigate to Edit Profile screen
2. **Tap Email**: Show email change flow (verify new email)
3. **Tap Change Password**: Show password change modal
4. **Tap Units**: Open units selection modal
5. **Select Unit**: Update preference, close modal
6. **Tap Daily Goal**: Open goal adjustment modal
7. **Adjust Slider**: Update goal value
8. **Tap Preset Goal**: Set goal to preset value
9. **Save Goal**: Update preference, close modal
10. **Tap Theme**: Open theme selection modal
11. **Select Theme**: Update theme, apply immediately, close modal
12. **Toggle Push Notifications**: Request permissions if enabling, update setting
13. **Tap Notification Settings**: Navigate to detailed notification settings
14. **Toggle Notification Type**: Update individual notification preference
15. **Tap Privacy Setting**: Open selection modal for that setting
16. **Change Privacy Setting**: Update preference, close modal
17. **Tap Terms/Privacy**: Open in-app browser or external browser
18. **Tap Sign Out**: Show confirmation dialog, sign out, navigate to auth

## Permission Handling

**Push Notifications:**
- Check permission status on toggle
- Request permission if not granted
- Show explanation if denied
- Link to system settings if permanently denied

**Theme Changes:**
- Apply immediately on selection
- Persist to AsyncStorage
- Update React Navigation theme
- Update status bar style

## Validation Rules

- Daily goal: 1,000 - 50,000 steps
- Password: Min 8 characters, requires letter and number
- Email: Valid email format, unique in system
