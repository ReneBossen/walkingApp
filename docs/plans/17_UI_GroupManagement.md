# Plan 17: Group Management UI

## Summary

Group Management screens enable users to create new groups, join existing groups through search or invite codes, configure group settings, manage members, and send invitations. These screens provide group administration capabilities for owners/admins.

## Screens

1. **Create Group** - Form to create a new group
2. **Join Group** - Search for groups or enter invite code
3. **Group Settings** - Configure group details (owner/admin only)
4. **Manage Members** - View and manage group members
5. **Invite Members** - Send invitations to friends

## Components

- `TextInput` (group name, description, search, invite code)
- `SegmentedButtons` (competition type selection)
- `Checkbox` (privacy settings)
- `Button` (create, join, save, invite)
- `Card` (group preview, member cards)
- `List.Item` (members list)
- `Menu` (member actions)
- `Searchbar` (member search)
- `Modal` (invite members)
- `IconButton` (add member, edit settings)

## API Calls

```typescript
POST /api/groups // Create new group
GET /api/groups/search?query={name} // Search public groups
POST /api/groups/{id}/join // Join public group
POST /api/groups/{id}/join-with-code // Join with invite code
PUT /api/groups/{id} // Update group settings
DELETE /api/groups/{id} // Delete group
GET /api/groups/{id}/members // Get all members
POST /api/groups/{id}/members/{userId}/promote // Promote to admin
POST /api/groups/{id}/members/{userId}/demote // Demote from admin
DELETE /api/groups/{id}/members/{userId} // Remove member
POST /api/groups/{id}/invite-code // Generate invite code
GET /api/groups/{id}/invite-code // Get current invite code
POST /api/groups/{id}/invitations // Invite friends
GET /api/groups/{id}/pending // Get pending member requests
PUT /api/groups/{id}/pending/{userId}/approve // Approve member
DELETE /api/groups/{id}/pending/{userId} // Deny member request
```

## State Management

**Local State:**
- `groupName: string`
- `description: string`
- `competitionType: 'daily' | 'weekly' | 'monthly'`
- `isPrivate: boolean`
- `requireApproval: boolean`
- `searchQuery: string`
- `selectedFriends: string[]`

**Global State:**
```typescript
interface GroupsStore {
  createGroup: (data: CreateGroupDto) => Promise<Group>;
  searchGroups: (query: string) => Promise<Group[]>;
  joinGroup: (groupId: string) => Promise<void>;
  joinWithCode: (code: string) => Promise<void>;
  updateGroup: (groupId: string, data: UpdateGroupDto) => Promise<void>;
  deleteGroup: (groupId: string) => Promise<void>;
  getMembers: (groupId: string) => Promise<GroupMember[]>;
  removeMember: (groupId: string, userId: string) => Promise<void>;
  promoteMember: (groupId: string, userId: string) => Promise<void>;
  approveMember: (groupId: string, userId: string) => Promise<void>;
  generateInviteCode: (groupId: string) => Promise<string>;
  inviteFriends: (groupId: string, friendIds: string[]) => Promise<void>;
}
```

## Interactions

1. **Create Group**: Fill form, validate, submit, navigate to group detail
2. **Join Public Group**: Search, tap result, confirm join
3. **Join with Code**: Enter code, validate, join group
4. **Edit Settings**: Owner/admin only, update fields, save changes
5. **Delete Group**: Owner only, show confirmation dialog, delete and navigate away
6. **View Members**: List all members with role badges
7. **Tap Member Menu**: Show options (Promote, Demote, Remove)
8. **Remove Member**: Admin/owner only, show confirmation, remove
9. **Approve/Deny Pending**: Review pending requests, approve or deny
10. **Generate Invite Code**: Create new code, display with copy option
11. **Share Invite**: Open native share sheet with deep link
12. **Invite Friends**: Select friends, send group invitations

## Validation Rules

- Group name: Required, 3-50 characters
- Description: Optional, max 500 characters
- Competition type: Required, one of daily/weekly/monthly
- Invite code: 6-12 characters, alphanumeric
- Owner cannot leave group (must transfer or delete)
- Cannot remove self unless admin removing themselves

## Acceptance Criteria

- [ ] Can create new group with valid data
- [ ] Can search for public groups
- [ ] Can join public group
- [ ] Can join private group with invite code
- [ ] Owner/admin can access settings
- [ ] Can update group details
- [ ] Owner can delete group with confirmation
- [ ] Can view all members with roles
- [ ] Admin can promote/demote members
- [ ] Admin can remove members
- [ ] Admin can approve/deny pending requests
- [ ] Can generate and copy invite code
- [ ] Can share invite link via native share
- [ ] Can invite friends from friend list
- [ ] Validation errors displayed clearly
- [ ] Real-time member list updates
- [ ] Permission checks enforced (owner/admin actions)
